# Diagnostic Guide: 166 Imported Clubs Not Visible

## Issue Summary
After importing 166 clubs via Excel, they show as successfully imported but are invisible to:
- Dashboard club list
- Publish Catalog (shows 0 clubs)
- Announcements manager

## Root Cause Analysis

The system has two data flows:
1. **Public View** → Uses snapshot (clubs-snapshot.json in Storage)
2. **Admin Registrations** → Uses Postgres (club_registrations table)
3. **Dashboard** → Reads from snapshot (generated by "Publish Catalog")

**When you import clubs via Excel:**
- Excel data is inserted into Postgres `club_registrations` table
- BUT the snapshot is NOT automatically regenerated
- So public catalog shows 0 clubs until you click "Publish Catalog"

**Problem:** Even after clicking "Publish Catalog", it shows 0 clubs
- This means publish-catalog query is finding 0 registrations
- Either:
  1. The 166 registrations weren't actually inserted into Postgres
  2. They were inserted but with wrong collection_id
  3. They were inserted but with wrong status  
  4. The query filter isn't working properly

## Diagnostic Steps

### Step 1: Check Import Response
When you import Excel, look at the API response for:
```json
{
  "verification": {
    "totalInCollection": 166,  // Should show 166
    "approvedInCollection": 166,  // Should show 166
    "searchedCollectionId": "collection-id-123"  // Note the ID
  }
}
```

**If totalInCollection is 0:** The registrations weren't saved to Postgres, or saved with wrong collection_id.

### Step 2: Use Diagnostic Endpoint
Call `/api/admin/registration-diagnostics` with your admin API key:

```bash
curl -H "x-admin-key: YOUR_API_KEY" \
  https://your-site.com/api/admin/registration-diagnostics
```

**Response shows:**
- How many registrations exist **per collection**
- Total counts by status (approved/pending/rejected)

**What to look for:**
- If a collection shows 0 approved registrations, but you just imported 166
  - Registration filter might be broken
  - OR they were inserted to wrong collection

### Step 3: Check Publish Catalog Logs
When you click "Publish Catalog", the server logs will show:
```
[Publish Catalog] Found X approved registrations
[Publish Catalog] DEBUG: Total registrations in DB: Y
```

**What to look for:**
- If total is > 0 but filtered to 0, there's a filter mismatch
- If total is 0, nothing is in the database

### Step 4: Manual Verification (Advanced)
If you have Supabase access:
1. Go to Supabase dashboard → SQL Editor
2. Run: 
   ```sql
   SELECT collection_id, status, COUNT(*) FROM club_registrations 
   GROUP BY collection_id, status
   ```
3. Verify 166 rows exist with correct collection_id and status='approved'

## Potential Causes & Fixes

### Cause 1: Wrong Collection ID
**Symptom:** Import says "success" but registers show in wrong collection

**Fix:** Check that collection ID in import dropdown matches what the import endpoint receives
- Look at `searchedCollectionId` in verification response

### Cause 2: Batch Insert Failed Silently  
**Symptom:** Some registrations worked but import reported fewer than 166

**Check:** Look at `successCount` vs `totalProcessed` in response
- If successCount < 166, some inserts failed
- Check error messages in response

### Cause 3: Status Not Set to 'approved'
**Symptom:** Registrations exist but status != 'approved'

**Check:** In diagnostic endpoint, look at status breakdown
- All imported clubs should have status='approved'

### Cause 4: Timestamp/Timezone Issue
**Symptom:** Registrations inserted but query returns 0

**Less likely** - but worth checking if diagnostic shows 0 total registrations

## Next Steps

1. **Attempt import again** with fresh Excel file
2. **Check verification response** - note the collection ID
3. **Call diagnostic endpoint** - see how many registrations per collection and status
4. **Click "Publish Catalog"** 
5. **Check server logs** for the publish-catalog operation
6. **Share results** of diagnostic endpoint

This will pinpoint exactly where the data loss is occurring.

## Performance Note

After a successful import, you MUST click "Publish Catalog" for changes to be visible publicly. The dashboard shows from the snapshot, not live registrations.

The import endpoint now:
- Verifies registrations were saved
- Reports what collection_id was used
- Shows filter results
- Returns all error messages

This should help diagnose where exactly the 166 clubs are being lost.
